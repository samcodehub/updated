Plyer Library in Python Apps: Cross-Platform Notifications & More

Python Plyer is a powerful and easy-to-use library that provides a unified way to access features on various platforms, such as notifications, vibrations, and battery information. It abstracts the platform-specific code and allows developers to create cross-platform applications effortlessly.

In this documentation, we will cover what Python Plyer is, how to install it, and how to use it to implement notifications.

What is Python Plyer?

Python Plyer is an open-source library that serves as an interface to access platform-specific features on Windows, macOS, Linux, Android, and iOS. It acts as a bridge between your Python code and the underlying operating system, allowing you to use various hardware and software functionalities in your applications without having to worry about platform-specific implementation details.

With Python Plyer, you can:

  • Display desktop notifications
  • Vibrate the device (if supported)
  • Get battery information
  • Access the flashlight (if available)
  • And more!

Installation

Before you can use Python Plyer, you need to install it. The easiest way is to use pip, the Python package manager. Open a terminal or command prompt and run the following command:

pip install plyer

This will download and install the Python Plyer library along with its dependencies.

Usage: Implementing Notifications

In this section, we will demonstrate how to use Python Plyer to implement desktop notifications for the top 5 news headlines from Google News.

# Import necessary libraries
from bs4 import BeautifulSoup
from urllib.request import urlopen
from plyer import notification
import time

# URL for Google News RSS feed
url = 'https://news.google.com/news/rss'

# Fetch the data from the URL
client = urlopen(url)
xml_data = client.read()
client.close()

# Parse the XML data
soup = BeautifulSoup(xml_data, 'xml')
news_list = soup.find_all('item')
news_list = news_list[0:5]

# Loop through the news list and show notifications
for news in news_list:
    # Display notification
    notification.notify(title="Today's News", message=news.title.text + '\nPublished on: ' + news.pubDate.text, timeout=20)
    
    # Wait for 20 minutes before showing the next notification
    time.sleep(1200)

The code above uses the BeautifulSoup library (learn more about BeautifulSoup library) to scrape the top 5 news headlines from Google News and then displays them as desktop notifications using Python Plyer's notification feature.

urllib.request (learn more about urllib.request module) is used to make HTTP requests to fetch the contents of the specified URL. In this code, it is used to open the Google News RSS feed and read its content.

The code uses time module (learn more about time module) to set a time interval to show the news. It waits for 20 minutes (1200 seconds) between each notification using time.sleep(1200).

Where to Use Python Plyer?

Python Plyer can be used in a wide range of applications, including but not limited to:

  • Desktop applications: You can use Plyer to provide notifications, battery status, and other platform-specific features to your Python desktop applications.
  • Mobile applications: When developing cross-platform mobile apps with frameworks like Kivy, Python Plyer can help you access device features like vibrations and flashlight functionality.
  • Automation scripts: Plyer can be used in automation scripts to provide real-time notifications and status updates.

Example Project

To illustrate the potential of Python Plyer, let's create a simple desktop news notification application using the Tkinter library (learn more about Tkinter library)for the user interface and Python Plyer for notifications.

import tkinter as tk
from bs4 import BeautifulSoup
from urllib.request import urlopen
from plyer import notification
import time

def fetch_news():
    url = 'https://news.google.com/news/rss'
    client = urlopen(url)
    xml_data = client.read()
    client.close()

    soup = BeautifulSoup(xml_data, 'xml')
    news_list = soup.find_all('item')
    news_list = news_list[0:5]

    for news in news_list:
        notification.notify(title="Today's News", message=news.title.text + '\nPublished on: ' + news.pubDate.text, timeout=20)
        time.sleep(1200)

if __name__ == "__main__":
    app = tk.Tk()
    app.title("News Notifier")

    tk.Label(app, text="Click the button to receive top news notifications").pack(pady=10)
    tk.Button(app, text="Fetch News", command=fetch_news).pack()

    app.mainloop()

In this example, we have created a simple Tkinter-based GUI application. When the user clicks the "Fetch News" button, it triggers the fetch_news() function, which retrieves the top 5 news headlines from Google News and displays them as desktop notifications using Python Plyer's notification feature.

Never Run Out of Juice: A Battery Notification Project using Python plyer & psutil

In today's fast-paced digital world, staying connected and productive relies heavily on our devices, and one of the most crucial aspects of keeping these devices operational is ensuring they have enough battery power. How often have you found yourself engrossed in your work or entertainment only to be interrupted by the dreaded low battery notification? To tackle this issue and keep you in the loop, we present a nifty Python project that will help you stay on top of your device's battery life.

Project Overview

In this blog, we'll delve into a simple yet effective project that notifies you about your device's battery percentage at regular intervals. We'll achieve this using the psutil and plyer libraries, which enable us to monitor the battery status and display notifications on our system.

Getting Started

To begin, make sure you have the psutil and plyer libraries installed. You can install them using the following pip commands:

pip install psutil
pip install plyer

Once the libraries are set up, you can create a Python script with the code provided above. Let's break down the key components of the project and understand how it works.

The Code Explained

The project revolves around a while loop that continuously monitors the battery status and sends notifications at regular intervals. Here's how the code works:

  1. Import Required Libraries:
    We start by importing the necessary libraries: psutil for battery monitoring and plyer for sending notifications.
  2. Conversion Function:
    The conversion function takes the remaining time in seconds and converts it into hours, minutes, and seconds. This helps us present the time remaining in a user-friendly format.
  3. Main Loop:
    The heart of the project is the while True loop. This loop continuously checks the battery status using psutil.sensors_battery() and calculates the time remaining using the conversion function. It then sends a notification using notification.notify() from the plyer library. The notification includes the remaining time and the battery percentage.
  4. Sleep Timer:
    After sending a notification, the script sleeps for an hour (60 minutes * 60 seconds). This means that you will receive a battery notification every hour, keeping you informed about your device's battery status without overwhelming you with notifications.
import psutil
from plyer import notification
import time

def conversion(sec): # function to convert seconds to minutes or hour
    sec_value = sec%(24*3600)
    hour_value = sec_value // 3600
    sec_value %= 3600
    min_value = sec_value// 60
    sec_value %= 60
    return hour_value, min_value

while True:
    battery = psutil.sensors_battery()
    h,m = conversion(battery.secsleft)
    notification.notify(
        title = "Battery Percentage",
        message =f'{h}Hr {m}Min {battery.percent}% remaining',
        timeout = 10 
    )
    time.sleep(60*60)  # notify after every 1 hour

By combining these components, you create a battery notification system that works quietly in the background, ensuring you're always aware of your device's battery status. This project is especially useful for those who are often engrossed in their work and might forget to keep an eye on their device's battery level.
